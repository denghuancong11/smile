# Agent失效分类与扰动

## 失败轨迹

```text
任务与环境：
预期行为：
实际行为：
首次偏离位置：
当时可用上下文：
缺失或冲突信息：
已执行动作：
验证输出：
后续影响：
可复现条件：
```

## 分类

| 类别 | 可观察现象 | 优先检查 |
|---|---|---|
| 目标漂移 | 后续改动偏离原始目标 | 目标、验收和阶段是否外部化 |
| 状态丢失 | 重复工作、忘记决定 | 状态记录、压缩和恢复协议 |
| 上下文遗漏 | 忽略关键文件或约束 | 来源地图、路由和上下文包 |
| 上下文噪声 | 被无关资料带偏 | 检索粒度、排序和裁剪 |
| 上下文过期 | 使用旧API或旧需求 | 来源版本、新鲜度和权威性 |
| 约束违反 | 越权、扩大范围、覆盖修改 | 行为契约和审批关卡 |
| 工具误用 | 用错工具、错误参数、忽略输出 | 工具规则、错误检查和降级 |
| 过早完成 | 未验证即宣称完成 | 完成定义和证据要求 |
| 修复震荡 | 多轮改动但错误不收敛 | 假设、错误分类和循环上限 |
| 范围膨胀 | 顺手重构或增加无关功能 | 原子任务和允许修改范围 |
| 协作冲突 | 并行任务相互覆盖 | 依赖图、所有权和合并顺序 |

一次失败可以有多个标签，但要找出首次导致偏离的机制。后续症状不自动等于根因。

## 四层诊断

```text
1. 表面失效是什么？
2. 直接触发条件是什么？
3. 哪个状态、上下文、任务、工具或验证机制失效？
4. 应建立什么可验证能力，而不是添加什么强调语？
```

为根因假设写出反证条件。若另一个机制能产生相同现象，设计区分实验。

## 扰动目录

- 删除一项非关键或关键上下文；
- 注入互相冲突的要求；
- 增加无关长文档；
- 提供过时设计记录；
- 模拟工具超时、权限不足或错误输出；
- 让测试在不同阶段失败；
- 在脏工作区中执行；
- 中断任务并从状态恢复；
- 修改一个外部依赖后继续；
- 并行处理共享文件或接口。

扰动必须对应真实风险。不要制造与使用场景无关的噪声，只为了得到失败案例。
