# smile

## Agent工作流工程

这是一套帮助AI稳定完成复杂任务的工作方法。

它要求AI先把目标、限制和完成标准说清楚，再查找过去踩过的坑，然后提出选择、主动寻找问题，由人做最终决定。决定后先小范围试验，确认有效再继续。每一步没有完成，就不能进入下一步。

## 能解决什么问题

这个 Skill 主要解决以下痛点：

1. AI没搞清需求就开始干  
   强制先确认真正目标、限制条件和完成标准，避免做出来的东西“看起来正确，但不是你要的”。

2. AI执行大任务时容易跑偏  
   把复杂任务分阶段完成。当前一步没有完成并检查通过，就不能进入下一步。

3. AI容易提出正确但普通的方案  
   要求AI从不同角度寻找容易被忽略的问题，而不是只给几个换了说法的常规方案。

4. AI只会证明自己的方案正确  
   强制它主动寻找反例、冲突和失败证据，减少自圆其说。

5. AI替人做了不该做的决定  
   AI负责收集、分析、提出选择和执行；目标选择、方案选择及风险范围必须由人决定。

6. 方案未经验证就直接全面实施  
   先做最小规模试验，确认核心想法成立后再继续投入。

7. 出错后只会不断修改  
   要求区分四种原因：需求没对齐、想法有问题、执行有问题、证据有问题，避免无方向地反复修改。

8. 中断后忘记做到哪里  
   工作状态被单独保存。重新开始时先读取已有进度，不会依靠AI自己回忆。

9. 相同错误反复出现  
   每次完成任务后保存错误、决定、经验、证据和适用范围。下一次遇到类似任务时先查这些记录。

10. 改进了一个问题，却破坏其他能力  
    每次改变工作方法后，都要用正常任务、历史失败任务、新任务和意外情况重新检查。

11. AI浪费大量内容和沟通成本  
    只读取当前阶段需要的信息，只解决当前缺口，减少重复分析、重复读取和无效输出。

12. 项目资料分散，AI找不准重点  
    先整理项目中的说明、设置、测试和关键文件，再决定需要读取什么，避免把整个项目全部塞给AI。

13. “故事讲得好”，但证据对不上  
    检查问题是否真实、为什么重要、方法解决了什么、证据支持什么，以及结论可以说到什么程度。

它不能直接解决的是：缺少真实资料、工具没有权限、用户不愿做决定、验收标准无法确定。遇到这些情况时，它会停止推进并指出缺少什么，而不是让AI猜测。

## 工作顺序

1. 说清楚要解决什么、为什么做、有哪些限制、怎样算完成；
2. 查找以前遇到过的相似问题；
3. 提出多个可能被忽略的方向；
4. 主动寻找反例和不成立的证据；
5. 由人选择目标、办法和风险范围；
6. 先做最小范围试验；
7. 分清是目标、判断、执行还是资料出了问题；
8. 保存经验和适用范围；
9. 把关键错误变成以后能自动发现的检查；
10. 下一次任务开始时先查找这些记录。

## 安装

1. 下载或克隆本仓库；
2. 将 `skill/engineer-agent-workflows` 文件夹复制到个人 Codex 技能目录；
3. Windows 默认目录为 `C:\Users\你的用户名\.codex\skills\engineer-agent-workflows`；
4. 重新打开 Codex。

## 使用

在Codex中输入：

```text
使用 $engineer-agent-workflows 帮我处理这个任务：<你的任务>
```

AI会从第一步开始。当前步骤没有完成时，它会停留并说明缺少什么，不会直接跳到后面的实施阶段。

## 已验证范围

- 50个自动检查通过；
- 缺少必要信息时不能进入下一步；
- 用户要求跳过流程时仍会被拦截；
- 手工伪造、放错步骤或被修改的检查记录会被拒绝；
- 中断后可以依据保存的状态继续。

它不能代替人的最终判断，也不能保证所有任务永远不出错。它的作用是让错误更早暴露、让重要决定由人确认，并让已经发现的问题不再只靠记忆避免。
